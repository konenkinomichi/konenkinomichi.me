❌ Google Sites — O que NÃO funciona

O Google Sites não aceita HTML/CSS/JS personalizado completo. Só permite embeber pequenos snippets via <iframe>, e mesmo assim com muitas restrições.

✅ Alternativas gratuitas (muito melhores para este caso)

  1. Vai a netlify.com/drop

  3. Recebes um link público imediatamente, por exemplo: https://konenkinomichi.netlify.app
  4. Podes personalizar o nome do domínio gratuitamente

🥈 GitHub Pages — gratuito e profissional

  1. Cria conta em github.com
  2. Cria um repositório novo
  3. Faz upload do ficheiro HTML com o nome index.html
  4. Vai a Settings → Pages → Deploy from branch

  1. Vai a tiiny.host
  2. Carrega o ficheiro HTML

Aqui está o passo a passo completo para publicares o teu site no GitHub:

1. Criar conta no GitHub

  4. Confirma o email que recebes na caixa de correio

2. Criar um repositório novo

  1. Depois de fazer login, clica no "+" no canto superior direito
  2. Escolhe "New repository"
  3. Em Repository name escreve: konenkinomichi
  4. Deixa em Public
  5. Clica em "Create repository"

3. Fazer upload do ficheiro HTML

  1. Na página do repositório que acabaste de criar, clica em "uploading an existing file"
  2. Arrasta o ficheiro konenkinomichi.html para a área indicada
  3. Renomeia o ficheiro para index.html — isto é importante!
  4. Em baixo, na área "Commit changes", clica no botão verde "Commit changes"

4. Ativar o GitHub Pages

  1. No teu repositório, clica em "Settings" (barra do topo)
  2. No menu lateral esquerdo clica em "Pages"
  3. Em "Branch" seleciona "main"
  4. Clica em "Save"

5. O teu site está online! 🎉

Após 1-2 minutos aparece o link: https://teu-username.github.io/konenkinomichi
